What’s new in BMS v25?

  • Enhanced password strength requirements: Strong password practices can be enforced in order to further secure BMS user accounts. Password length and complexity rules are customizable based on specific organizational policies.

  • Reinforced privacy of germplasm details used in a study: Administrators can control who has access to the Germplasm Details popup within the Study Manager.

  • Additional batch capabilities in Study Manager: Batch import of environment-level details and conditions via file, batch deletion of incorrect observation values, and the application of ’Accept’ and ’Discard’ actions on pending data per environment are now possible!

  • Transposed data collection file for sub-observations: Sub-observation units such as individual plants are represented as columns in an Excel studybook file to facilitate data collection and import.

  • Germplasm descriptors inherited by next-generation advances: Attributes and passport descriptors can be optionally propagated to the next-generation germplasm created through a study ’advance’.

  • Improvements to interoperability with KSU Fieldbook using BrAPI 2.0
